# KalmanFilter_jax.py における共分散更新の数値的安定化リファクタリング

## 背景：なぜ最適化が不安定だったのか？
`plot_tradeoff_jax.py` で状態遷移モデルのプロセスノイズ分散（`Q0_alpha` など）を周辺尤度最大化（最尤推定）で最適化する際、パラメータが下限（1E-12付近）に張り付いて最適化が不安定になる現象が確認されました。

これは「Process noise collapse（降水モデルによる説明能力が高く、状態変動ノイズが論理的に 0 とみなされた結果）」という統計モデリングとして正しい挙動ですが、コード内部の計算方法が原因で数値的な破綻・発散を引き起こしていました。

### 元々の実装（Information Filter 形式の Woodbury 公式）
元の `KalmanFilter_jax.py` では事後共分散や尤度計算の途中で、カルマンゲイン相当の量を求めるために**事前共分散行列 $P_t$ の逆行列を直接計算**していました。

```python
# [旧実装の抜粋]
safe_Pt = jnp.where(has_data, Pt, jnp.identity(2))
inv_Pt = jnp.linalg.inv(safe_Pt)  # ← 事前共分散行列の逆行列を陽に計算

Z2_reg = jnp.where(has_data, Z2, 0.0)
mid_matrix = Z2_reg/h0 + inv_Pt   # ← O(1)スケールの値と足し合わせ
mid_inv = jnp.linalg.inv(mid_matrix)
```

この実装では、$Q_0 \to 0$ に伴い $P_t$ が 0 に極めて近づいたとき（要素が $10^{-12}$ のオーダーなどに縮退したとき）、`inv_Pt` は $10^{12}$ のような**巨大な値に発散**します。
この巨大な行列と $O(1)$ スケールである `Z2_reg/h0` を足し合わせて再度逆行列を計算する過程で、深刻な情報落ち（丸め誤差）が発生し、カルマンゲインや周辺尤度に $\text{NaN}$ やノイズが無数に混入してしまっていました。

---

## 修正内容：行列の逆行列 ($P_t^{-1}$) を回避する数学的変形
この問題を根本から解決するため、数学的な恒等式を活用し、同じ結果を与えつつも **$P_t$ の逆行列を一切計算しない** 形にアップデートしました。

### 1. `mid_inv` の再定式化
元の `mid_inv` は以下の式を計算していました：
$$ \text{mid\_inv} = (P_t^{-1} + \frac{Z^T Z}{h_0})^{-1} $$

これを、同値な以下の式（Woodburyの恒等式変形）に書き換えました：
$$ \text{mid\_inv} = P_t (I + \frac{Z^T Z}{h_0} P_t)^{-1} $$

**実装例:**
```python
# [新実装の抜粋]
Z2_reg = jnp.where(has_data, Z2, 0.0)
X = Z2_reg / h0
inner_mat = jnp.identity(2) + X @ Pt      # ← Ptの直接の逆行列を取らない
inner_inv = jnp.linalg.inv(inner_mat)
mid_inv_raw = Pt @ inner_inv
mid_inv = (mid_inv_raw + mid_inv_raw.T) / 2.0  # 数学的な対称性の担保
```
この式であれば、$P_t \to 0$ と極限をとっても、括弧の逆行列計算部分は $(I + 0)^{-1} = I$ となり、破綻せず安全に計算が可能になります。最終的な `mid_inv` も緩やかに 0 に近づくだけです。

### 2. カルマンゲイン `Xi` と事後共分散 `Ptt` の導出単純化
元のコードでは $P_t^{-1}$ が現れずに見えていた更新式も、実は `mid_inv` （すなわち $P_{t|t}$）を用いることで直感的かつ安定して表現できます。

**更新式の対応：**
$$ P_{t|t} = (P_t^{-1} + \frac{Z^T Z}{h_0})^{-1} \equiv \text{mid\_inv} $$

カルマンゲイン相当の行列 $\Xi$ は本来の式を整理すると、事後共分散を用いてシンプルに表現できます。
$$ \Xi = P_{t|t} \frac{Z^T}{h_0} $$
(コード上では $Z^T$ や $Z$ の形状が予め畳み込まれているため、実質的に $\Xi_{\text{code}} = P_{t|t} / h_0$ 相当の扱いになります)

これらをまとめると、コードは以下のように極めてシンプルかつ堅牢になります。

```python
# [新実装の抜粋]
Xi = mid_inv / h0

# 旧: Ptt = jnp.where(has_data, Pt - Xi @ (Z2_reg @ Pt @ Z2_reg + h0 * Z2_reg) @ Xi.T, Pt)
# 新: 数学的に全く等価だが極限で破綻しない式
Ptt = jnp.where(has_data, mid_inv, Pt)
```

---

## 結論と効果
このアルゴリズムのリファクタリングにより：
1. **$P_t^{-1}$ に起因する発散が 100% 排除** されました。
2. これにより、物理的に妥当である「尤度の最大化における Process noise covariance のゼロへの縮退 ($Q \to 0$)」の限界領域に到達しても、アルゴリズムが安定して動作します。
3. 行列式の二次方程式への帰着（model size 化）などの既存の優秀な数学的最適化はそのまま保持されています。

結果として、`plot_tradeoff_jax.py` による最適化ルーチンが NaN による崩壊を起こさず、JAXの強力な自動微分の恩恵を安全に受けながら最後まで安定して最尤解を探索できるようになりました。
